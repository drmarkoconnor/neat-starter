Default layouts

This home page (index.html) references the default.html layout (in _layouts dir) in its page frontmatter which in turn injects the navbar and the footer via an include _partials/navbar and header.

Front matter conditonal logic

The template default page uses some conditional logic in its head to add a css link to bootstrap, font-awesome and for metadata depending on a particular page's frontmatter as follows.

  • description: "any text here gets into the page's meta data
    • " ( and can be used for other purposes too )
  • boot: true
    • will mean the page can reference bootstrap classes
  • awesome: true
    • will allow use of the "fa fas-name" icons from Font awesome

    Data derived nav links

    The navbar derives its links from iterating over the elements of the navigation.yaml file. The Netlify CMS can change this file too ( whetheer that is useful is a use case for each user of this template)

    CSS Styling

    The main style sheet is listed last in the default head so will have a greater priority thasn the others referenced.

    Data derived content

    This css styled 'table' (it's not a table ) is generated from data, held in the quicklinks.yaml file, iterated over by the partial contents.html file. Styling is in styles.css using a .quicklinks class

    Colour scheme

    Define var(--primary-color-XXX):hsl(x,y%,z%) and var(--secondary-color-XXX):hsl(x,y%,z%) using XXX to indicate different shades of same color. Makes colour ( I'm English, this is how we spell it!) palette very easy to change for whole site.
